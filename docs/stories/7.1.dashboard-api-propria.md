# Story 7.1 — Adaptar Dashboard para API Propria

## Status
Draft

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 2: Dashboard Proprio)

## Story
Como **gestor da corretora**, quero que o dashboard carregue dados do banco Neon ao inves do ClickUp, para ter carregamento rapido (< 2s vs 10-15s) e independencia da plataforma externa.

## Acceptance Criteria

- [ ] AC1: Dashboard principal carrega cotacoes via `GET /api/cotacoes` (Neon) ao inves do ClickUp API
- [ ] AC2: KPIs (A Receber, Valor Perda, Total, Conversao) calculados via `GET /api/kpis`
- [ ] AC3: Filtros de ano, mes, status e busca funcionam via query params na API propria
- [ ] AC4: Colunas Kanban (Atraso, Perda, Em Andamento, Fechado) renderizam com dados do Neon
- [ ] AC5: Breakdown de status funciona com dados do Neon
- [ ] AC6: Cards de cotacao incluem link para pagina de edicao `/cotacoes/[id]`
- [ ] AC7: View de Renovacao carrega cotacoes com `is_renovacao=true` do Neon
- [ ] AC8: Tempo de carregamento inicial < 2 segundos (vs 10-15s com ClickUp)
- [ ] AC9: Cotador logado ve apenas suas cotacoes no dashboard
- [ ] AC10: Admin ve todas as cotacoes
- [ ] AC11: Auto-refresh funcional com dados do Neon
- [ ] AC12: Graficos (Chart.js) renderizam com dados do Neon

## Tasks
