# Story 4.2 — Cards de Desempenho por Cotador

## Status
InProgress

## Executor
@dev (Dex)

## Story
Como **gestor da corretora**, quero ver cards individuais de cada cotador com foto, nome, metas individuais e métricas de desempenho (cotações realizadas, fechadas, em perda, % atingimento), para acompanhar a performance da equipe de forma visual.

## Acceptance Criteria

- [x] AC1: Seção "Desempenho por Cotador" dentro da view Metas
- [x] AC2: Card por cotador com: foto (ou avatar placeholder), nome, cargo
- [x] AC3: Cada card exibe métricas: total cotações, fechadas, em perda, outros status
- [x] AC4: Meta individual editável por cotador (R$ e/ou qtd cotações)
- [x] AC5: Barra de progresso individual com % de atingimento
- [x] AC6: Cotadores identificados automaticamente pelo campo `assignees` das tasks do ClickUp
- [x] AC7: Cards ordenados por % de atingimento (melhor performance primeiro)
- [x] AC8: Filtro de mês se aplica aos cards
- [x] AC9: Dados de perfil (foto, nome) persistidos em localStorage
- [x] AC10: Funciona com dados pré-carregados (offline) — sem fotos, apenas nomes

## Tasks

- [x] Criar seção HTML `.cotador-cards` dentro da view Metas
- [x] Implementar função `extractCotadores(tasks)` que agrupa tasks por assignee
- [x] Criar card component com layout foto + nome + métricas + barra de progresso
- [x] Implementar modal/inline edit para meta individual e dados de perfil (foto URL, nome)
- [x] Persistir perfis e metas individuais em localStorage

## Dev Notes

- Assignees no ClickUp vêm como `task.assignees[{username, profilePicture}]`
- `profilePicture` pode servir como foto se disponível, senão usar avatar com iniciais
- Usernames conhecidos no workspace: "SAUDEEODONTO", "RAMOS ELEMENTARES" (podem ser equipes, não pessoas)
- Pode ser necessário um mapeamento manual username → nome real (configurável no localStorage)
- Formato localStorage: `apolizza_cotadores` → JSON `[{ username, displayName, photoUrl, metaValor, metaQtd }]`
