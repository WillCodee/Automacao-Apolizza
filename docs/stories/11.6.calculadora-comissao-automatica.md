# Story 11.6 — Calculadora Automatica de Comissao

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-002 — Apolizza CRM v2 (Epic 2: Upgrade CRM Completo)

## Story
Como **cotador**, quero que o sistema calcule automaticamente a comissao e o valor a receber quando eu preencher o premio e a porcentagem de comissao, para evitar erros de calculo e agilizar o preenchimento.

## Acceptance Criteria

- [x] AC1: Novo campo `comissao_percentual` (%) no formulario de cotacao
- [x] AC2: Ao preencher `premio_sem_iof` + `comissao_percentual`, o campo `comissao` (R$) calcula automaticamente
- [x] AC3: `a_receber` calcula como `comissao` (ou formula customizavel pelo admin)
- [x] AC4: Calculo acontece em tempo real no frontend (onChange)
- [x] AC5: Valores calculados podem ser editados manualmente (override)
- [x] AC6: Se o usuario editar manualmente, o auto-calculo desativa para aquele campo
- [x] AC7: Tabela de comissao por seguradora (admin configura %) (comissao_tabela table + API /api/comissao-tabela)
- [x] AC8: Sugestao automatica de % quando seguradora e selecionada (auto-suggestion via useEffect when seguradora selected)

## Tasks

- [x] Adicionar coluna `comissao_percentual DECIMAL(5,2)` na tabela cotacoes
- [x] Criar tabela `comissao_tabela` (id, seguradora, produto, percentual, vigencia_inicio, vigencia_fim)
- [x] Criar endpoint `GET /api/comissao-tabela` para consultar % por seguradora/produto
- [x] Criar endpoint `POST /api/comissao-tabela` (admin cadastra tabelas)
- [x] Atualizar `cotacao-form.tsx`: adicionar campo % e logica de calculo reativo
- [x] Implementar auto-calculo: `comissao = premio_sem_iof * (comissao_percentual / 100)`
- [x] Implementar sugestao de %: ao selecionar seguradora+produto, buscar na tabela
- [x] Atualizar validations.ts com novo campo
- [x] Gerar migration para novas colunas/tabela
- [x] Testar calculos com valores reais
