# Story 10.4 — Transacoes DB e Correcoes de Seguranca

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-002 — Apolizza CRM v2 (Epic 1: Hardening & Seguranca)

## Story
Como **admin do sistema**, quero que operacoes multi-step no banco usem transacoes e que falhas de seguranca conhecidas estejam corrigidas, para garantir consistencia de dados e protecao contra ataques.

## Acceptance Criteria

- [x] AC1: PUT `/api/cotacoes/[id]` envolve update + history insert em uma unica transacao DB
- [x] AC2: Se o insert de historico falhar, o update da cotacao faz rollback
- [ ] AC3: DELETE `/api/cotacoes/[id]` envolve soft delete + history em transacao
- [x] AC4: CRON_SECRET undefined retorna 500 ("nao configurado") — ja estava implementado
- [x] AC5: Upload de docs sanitiza nomes removendo pontos iniciais (`.env`, `.htaccess`)
- [x] AC6: Erros de Vercel Blob sao logados (nao silenciados) no delete de docs
- [x] AC7: Headers de seguranca configurados no next.config: X-Frame-Options, X-Content-Type-Options, Referrer-Policy, HSTS, Permissions-Policy, X-XSS-Protection

## Tasks

- [x] Implementar `db.transaction()` no PUT `/api/cotacoes/[id]/route.ts`
- [ ] Implementar `db.transaction()` no DELETE `/api/cotacoes/[id]/route.ts`
- [x] CRON_SECRET: check `if (!cronSecret)` ja existia — mantido
- [x] Fix sanitizacao de nome de arquivo: adicionado `.replace(/^\.+/, "")` + validacao nome vazio
- [x] Trocar `catch {}` silencioso por `catch (blobError) { console.error(...) }` no delete de blob
- [x] Adicionar `headers` no `next.config.ts` com 6 headers de seguranca
