# Story 5.2 — Autenticacao: Auth.js v5 + Login + Roles + Middleware

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 0: Fundacao)

## Story
Como **gestor da corretora**, quero que o sistema tenha autenticacao com email/senha e dois niveis de acesso (admin e cotador), para que cada usuario veja apenas o que lhe compete e os dados estejam protegidos.

## Acceptance Criteria

- [x] AC1: Auth.js v5 configurado com Credentials Provider (email + senha)
- [x] AC2: JWT strategy (sem Drizzle Adapter — Credentials nao usa sessions table)
- [x] AC3: JWT strategy ativo com role do usuario no token
- [x] AC4: Proxy (Next.js 16) protege todas as rotas exceto `/login` e `/api/auth/*`
- [x] AC5: Usuario nao autenticado e redirecionado para `/login` (307)
- [x] AC6: Sessao inclui `user.role` (admin ou cotador) e `user.id`
- [x] AC7: 3 usuarios admin criados via seed (senha bcrypt salt 12)
- [x] AC8: Funcao helper `getCurrentUser()` disponivel para API routes e Server Components
- [x] AC9: Funcao helper `requireAdmin()` que retorna 403 se nao for admin
- [x] AC10: Logout funcional com redirect para `/login` (SignOutButton)
- [x] AC11: Senhas armazenadas com bcrypt (salt rounds = 12)

## Dev Notes

- **Auth.js v5** usa a nova API `NextAuth()` que exporta `{ handlers, auth, signIn, signOut }`
- **Credentials Provider:** Unica opcao necessaria — login com email + senha
- **JWT:** Preferido sobre database sessions para serverless (sem lookup de sessao a cada request)
- **Middleware matcher:** `export const config = { matcher: ['/((?!login|api/auth|_next/static|_next/image|favicon.ico).*)'] }`
- **Seed admins:** Os 3 emails/senhas serao definidos pelo cliente. Usar bcrypt com `saltRounds: 12`
- **Depende de:** Story 5.1 (schema users deve existir)
