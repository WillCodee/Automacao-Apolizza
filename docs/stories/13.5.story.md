### Observações Técnicas

- Log assíncrono com try-catch isolado (não bloqueia operação principal se falhar)
- JSONB no PostgreSQL permite queries futuras em campos específicos de `detalhes`
- Índice composto (tarefa_id, created_at) otimiza query principal
- Detalhes formatados dinamicamente no frontend (não armazenado texto final no banco)
- Emoji icons (sem lib externa) para performance
- Timestamp relativo calculado no client (atualiza com refresh)

### Decisões de Design

- Audit log passivo (não bloqueia operações se falhar)
- Formatação de detalhes no frontend (flexibilidade futura)
- Timeline vertical simples (sem lib externa)
- EDITADA não implementada (rota PATCH /api/tarefas/[id] não existe - apenas sub-resources)
- Refresh manual via prop (não polling automático)
