# Story 8.3 — Audit Trail (Historico de Alteracoes)

## Status
Draft

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 3: Automacoes + Polish)

## Story
Como **gestor da corretora**, quero ver o historico completo de alteracoes de cada cotacao, para saber quem mudou o que e quando — garantindo rastreabilidade e transparencia.

## Acceptance Criteria

- [ ] AC1: Toda alteracao em campos da cotacao gera registro em `cotacao_history`
- [ ] AC2: Registro inclui: cotacao_id, user_id (quem alterou), field_name, old_value, new_value, changed_at
- [ ] AC3: Alteracoes automaticas (cron/sistema) registram user_id como NULL
- [ ] AC4: Endpoint `GET /api/cotacoes/[id]/history` retorna historico da cotacao
- [ ] AC5: Historico ordenado por data (mais recente primeiro)
- [ ] AC6: Secao "Historico" visivel na pagina de edicao da cotacao
- [ ] AC7: Timeline visual com icones por tipo de alteracao (status, valor, data)
- [ ] AC8: Somente admin e o cotador responsavel veem o historico

## Tasks

- [ ] Implementar funcao `recordHistory(cotacaoId, userId, changes[])` no backend
- [ ] Integrar `recordHistory` no endpoint `PUT /api/cotacoes/[id]` — comparar old vs new
- [ ] Criar endpoint `GET /api/cotacoes/[id]/history`
- [ ] Criar componente `CotacaoHistory` com timeline visual
- [ ] Integrar componente na pagina de edicao `/cotacoes/[id]`
- [ ] Implementar controle de acesso ao historico
- [ ] Testar que edicoes geram registros corretos
- [ ] Testar que mudancas de status pelo cron geram registros com user_id NULL

## Dev Notes

- **Diff de campos:** No PUT, buscar cotacao atual, comparar campo a campo com o body, gerar array de `{ field_name, old_value, new_value }` para cada diferenca
- **Performance:** Inserir historico em batch (um INSERT com multiplas rows)
- **Timeline component:** Listar registros com icone, data, "usuario alterou campo de X para Y"
- **Depende de:** Story 5.1 (tabela cotacao_history), Story 5.3 (endpoint PUT)
