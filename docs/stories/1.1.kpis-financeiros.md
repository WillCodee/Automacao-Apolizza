# Story 1.1 — KPIs Financeiros no Topo do Dashboard

## Status
InProgress

## Executor
@dev (Dex)

## Story
Como **gestor da corretora**, quero ver cards de KPIs financeiros no topo da view Cotações (A Receber, Valor em Perda, Total Cotações, Taxa de Conversão), para ter visão imediata da saúde financeira sem precisar navegar entre tabelas.

## Acceptance Criteria

- [x] AC1: Card "A Receber" exibe soma de `FIELD_A_RECEBER` de todas as tasks com status "fechado", filtrado por ano/mês selecionado
- [x] AC2: Card "Valor em Perda" exibe soma de `FIELD_VALOR_PERDA` de todas as tasks com status "perda", filtrado por ano/mês selecionado
- [x] AC3: Card "Total Cotações" exibe contagem total de tasks no período filtrado
- [x] AC4: Card "Taxa de Conversão" exibe percentual (fechadas / total * 100) com 1 casa decimal
- [x] AC5: Cards atualizam automaticamente quando filtros de ano/mês são alterados
- [x] AC6: Cards atualizam junto com o auto-refresh de 5 minutos
- [x] AC7: Valores financeiros formatados em BRL (R$ X.XXX,XX)
- [x] AC8: Layout responsivo — cards empilham em telas menores que 768px
- [x] AC9: Dados pré-carregados (PRELOADED) alimentam os KPIs quando offline

## Tasks

- [x] Criar seção HTML `.kpi-strip` com 4 cards acima da tabela de cotações
- [x] Implementar função `calcKPIs(tasks)` que extrai valores dos custom fields
- [x] Integrar cálculo no fluxo de `renderFiltered()` e `loadData()`
- [x] Adicionar formatação BRL com `Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' })`
- [x] Adicionar CSS responsivo para os cards KPI
- [x] Testar com dados pré-carregados (offline) e dados da API (online)
