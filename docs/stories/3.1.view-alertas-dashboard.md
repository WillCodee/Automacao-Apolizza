# Story 3.1 — View de Alertas no Dashboard

## Status
InProgress

## Executor
@dev (Dex)

## Story
Como **gestor da corretora**, quero uma nova view "Alertas" na sidebar do dashboard que mostre tasks atrasadas, vencendo hoje e vencendo nos próximos 7 dias, para ter um painel de urgências centralizado.

## Acceptance Criteria

- [x] AC1: Nova opção "Alertas" na sidebar com ícone de sino (🔔)
- [x] AC2: Seção "Atrasados" — lista tasks com due_date < hoje E status != "fechado", ordenadas por dias de atraso (mais atrasada primeiro)
- [x] AC3: Seção "Vencendo Hoje" — lista tasks com due_date = hoje E status != "fechado"
- [x] AC4: Seção "Próximos 7 Dias" — lista tasks com due_date entre amanhã e +7 dias E status != "fechado"
- [x] AC5: Cada task exibe: nome (link para ClickUp), responsável, data, dias de atraso/para vencer, prioridade
- [x] AC6: Contadores no topo de cada seção (ex: "3 atrasados", "2 vencendo hoje")
- [x] AC7: Badge com número total de alertas no item da sidebar
- [x] AC8: View atualiza junto com auto-refresh e filtros de ano
- [x] AC9: Destaque visual diferenciado — vermelho (atrasados), amarelo (hoje), azul (próximos)

## Tasks

- [ ] Adicionar item "Alertas" na sidebar com badge de contagem
- [ ] Criar HTML da view `.view-alertas` com 3 seções colapsáveis
- [ ] Implementar função `calcAlerts(tasks)` que categoriza tasks por urgência
- [ ] Criar cards de alerta com link direto para task no ClickUp
- [ ] Adicionar CSS com cores por nível de urgência
- [ ] Integrar com sistema de navegação existente (showView)
- [ ] Garantir funcionamento com dados pré-carregados (offline)

## Dev Notes

- Sistema de navegação existente usa `showView(viewName)` — seguir mesmo padrão
- Sidebar items usam classe `.nav-item` com `.active` para selecionado
- `due_date` vem como timestamp ms do ClickUp — comparar com `Date.now()`
- Cálculo de "dias de atraso": `Math.floor((Date.now() - due_date) / 86400000)`
- Branch: `feat/dashboard-v2-modulos`
