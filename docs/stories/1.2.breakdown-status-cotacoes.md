# Story 1.2 — Breakdown de Cotações por Status

## Status
InProgress

## Executor
@dev (Dex)

## Story
Como **gestor da corretora**, quero ver um breakdown visual de cotações separadas por status (fechadas, perda, em andamento, pendência, não iniciado, atrasado), para entender rapidamente a distribuição do pipeline comercial.

## Acceptance Criteria

- [x] AC1: Seção de breakdown exibe contagem de tasks por cada status existente no ClickUp
- [x] AC2: Cada status tem cor correspondente (verde=fechado, vermelho=perda, amarelo=em andamento, cinza=não iniciado, laranja=atrasado, azul=pendência)
- [x] AC3: Breakdown respeita filtros de ano/mês ativos
- [x] AC4: Exibe porcentagem de cada status em relação ao total
- [x] AC5: Atualiza junto com o auto-refresh e mudança de filtros
- [x] AC6: Funciona com dados pré-carregados (offline)

## Tasks

- [x] Criar seção HTML `.status-breakdown` abaixo dos KPIs e acima da tabela
- [x] Implementar função `calcStatusBreakdown(tasks)` que agrupa tasks por `task.status.status`
- [x] Mapear cores por status (reutilizar o padrão visual existente)
- [x] Adicionar barra horizontal ou pills com contagem + percentual
- [x] Integrar no fluxo de `renderFiltered()` existente

## Dev Notes

- Status disponíveis no ClickUp da Apolizza: "não iniciado", "em andamento", "pendencia", "fechado", "perda", "atrasado"
- O campo `task.status.status` retorna string lowercase
- Já existe `PRIORITY_MAP` — criar equivalente `STATUS_MAP` com cores e labels
- Branch: `feat/dashboard-v2-modulos`
