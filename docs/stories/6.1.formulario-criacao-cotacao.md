# Story 6.1 — Formulario de Criacao de Cotacao

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 1: CRUD + Migracao)

## Story
Como **cotador**, quero um formulario completo para criar novas cotacoes com todos os 19 campos do seguro, para registrar cada oportunidade de venda diretamente no sistema sem depender do ClickUp.

## Acceptance Criteria

- [x] AC1: Pagina `/cotacoes/new` com formulario de criacao
- [x] AC2: Campo "Nome do Cliente" (text, obrigatorio)
- [x] AC3: Campo "Status" (dropdown com 12 opcoes do workflow)
- [x] AC4: Campo "Prioridade" (dropdown: urgente, alta, normal, baixa)
- [x] AC5: Campo "Data Limite" (date picker)
- [x] AC6: Campo "Tipo Cliente" (dropdown: NOVO/CASA, NOVO, RENOVACAO)
- [x] AC7: Campo "Contato Cliente" (tel input, sem mask lib)
- [x] AC8: Campo "Seguradora" (text)
- [x] AC9: Campo "Produto" (dropdown com 49 opcoes)
- [x] AC10: Campo "Situacao" (dropdown: IMPLANTACAO, COTAR, CLIENTE, RAUT, FECHADO, PERDA/RESGATE)
- [x] AC11: Campo "Indicacao" (text)
- [x] AC12: Campos de data: Inicio Vigencia, Fim Vigencia, Primeiro Pagamento, Proxima Tratativa
- [x] AC13: Campos financeiros: Premio Sem IOF, Comissao, A Receber, Valor em Perda (number inputs)
- [x] AC14: Campo "Parcelado Em" (number, inteiro)
- [x] AC15: Campo "Observacao" (textarea)
- [x] AC16: Campo "Mes" (dropdown JAN-DEZ) e "Ano" (dropdown 2024-2027)
- [x] AC17: Toggle "Renovacao" (checkbox)
- [x] AC18: Cotador e automaticamente assignado como responsavel (API default)
- [x] AC19: Admin pode selecionar qualquer cotador como responsavel
- [x] AC20: Botao "Salvar" envia POST para `/api/cotacoes` e redireciona para lista
- [x] AC21: Feedback visual de sucesso (green) ou erro (red) inline
- [x] AC22: Formulario responsivo (grid cols 1/2/3)
