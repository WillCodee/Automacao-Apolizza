# Story 1.3 — Indicador Visual de Campos Incompletos

## Status
InProgress

## Executor
@dev (Dex)

## Story
Como **gestor da corretora**, quero ver um indicador visual nas tasks que possuem campos obrigatórios não preenchidos (conforme regras por status), para identificar rapidamente problemas de cadastro sem precisar abrir cada task no ClickUp.

## Acceptance Criteria

- [x] AC1: Tasks com status "não iniciado" sem os campos obrigatórios (Fim Vigência, Início Vigência, Indicação, Produto, Seguradora, Situação, Tipo de Cliente) exibem ícone de alerta
- [x] AC2: Tasks com status "fechado" sem os campos obrigatórios (Comissão, Primeiro Pagamento, A Receber, Parcelado Em, Prêmio Sem IOF) exibem ícone de alerta
- [x] AC3: Tasks com status "perda" sem o campo "Valor em Perda" exibem ícone de alerta
- [x] AC4: Tooltip no ícone lista quais campos estão faltando
- [x] AC5: Counter global no topo mostra "X tasks com campos incompletos"
- [x] AC6: Indicador é puramente visual — não bloqueia nada no dashboard

## Tasks

- [x] Mapear UUIDs dos custom fields obrigatórios por status (verificar no ClickUp quais IDs correspondem a cada campo)
- [x] Implementar função `checkRequiredFields(task)` que retorna array de campos faltantes
- [x] Adicionar ícone na linha/card da task quando houver campos faltantes
- [x] Implementar tooltip com lista dos campos ausentes
- [x] Adicionar counter global "X tasks com campos incompletos" na seção KPI
- [x] Integrar validação no fluxo de render

## Dev Notes

- Os campos obrigatórios por status estão definidos no briefing do cliente (Módulo 1)
- Será necessário mapear os nomes dos campos para seus UUIDs no ClickUp — podem não estar todos em `FIELD_*` constantes atuais
- Custom fields vêm em `task.custom_fields[]` — checar se `value` é null/undefined/empty
- Este é um proxy do Módulo 1 (campos obrigatórios) implementado como visibilidade, não como enforcement
- Branch: `feat/dashboard-v2-modulos`
