# Story 11.7 — Kanban Board de Cotacoes

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-002 — Apolizza CRM v2 (Epic 2: Upgrade CRM Completo)

## Story
Como **cotador**, quero ver minhas cotacoes em um board Kanban com colunas por status e poder arrastar para mudar status, para ter visao visual do pipeline e agilizar o workflow.

## Acceptance Criteria

- [x] AC1: Nova view "Kanban" na pagina de cotacoes (toggle Lista/Kanban)
- [x] AC2: Colunas do Kanban baseadas nos status ativos do `status_config`
- [x] AC3: Cada card mostra: nome, seguradora, valor a receber, dias no status
- [x] AC4: Drag-and-drop muda o status da cotacao (PUT /api/cotacoes/[id])
- [ ] AC5: Validacao de campos obrigatorios ao dropar em status terminal (fechado, perda)
- [x] AC6: Contagem de cards por coluna no header
- [x] AC7: Cores das colunas baseadas no `status_config.color`
- [x] AC8: Filtros de ano/mes/cotador aplicam no Kanban tambem
- [ ] AC9: Responsivo: em mobile, colunas viram accordion vertical

## Tasks

- [ ] Instalar `@dnd-kit/core` + `@dnd-kit/sortable` para drag-and-drop
- [ ] Criar componente `KanbanBoard` com colunas dinamicas baseadas em status_config
- [ ] Criar componente `KanbanCard` com info resumida da cotacao
- [ ] Implementar drag-and-drop com callback `onStatusChange(cotacaoId, newStatus)`
- [ ] No drop, validar campos obrigatorios antes de salvar (reutilizar status-validation.ts)
- [ ] Se validacao falha, mostrar modal com campos faltando e cancelar o drop
- [ ] Adicionar toggle "Lista | Kanban" no header da pagina de cotacoes
- [ ] Persistir preferencia de view no localStorage
- [ ] Implementar layout mobile (accordion)
- [ ] Testar com 50+ cotacoes para verificar performance do drag
