## Story
Como **gestor da corretora**, quero criar e gerenciar tarefas diárias para os cotadores, para organizar as atividades e garantir que nada seja esquecido.

### Decisões Técnicas

1. **Schema Design:**
   - Enum `tarefaStatusEnum` com 4 status: Pendente, Em Andamento, Concluída, Cancelada
   - Foreign keys com `onDelete: cascade` para integridade referencial
   - Índices em `status`, `cotadorId`, `criadorId`, `dataVencimento` para performance
   - Relations Drizzle com `relationName` para diferenciar cotador vs criador

2. **Permissões:**
   - `admin`: CRUD completo
   - `cotador`: Read-only (apenas suas tarefas)
   - Nota: "gestor" = "admin" neste contexto (enum só tem `["admin", "cotador"]`)

3. **API Routes:**
   - Next.js 16 async params: `params: Promise<{ id: string }>` (CRITICAL FIX)
   - Validação de permissões em todas as rotas
   - Uso de `apiSuccess()`, `apiError()`, `apiPaginated()` helpers
   - Filtragem automática: cotadores veem apenas suas tarefas

4. **Frontend:**
   - Cards Grid (responsivo: 1/2/3 colunas)
   - Filtros por status (Todas/Pendentes/Em Andamento/Concluídas)
   - Modal de criação (apenas admin)
   - Badge de status com cores distintas
   - Indicação visual de tarefas atrasadas (borda vermelha)
