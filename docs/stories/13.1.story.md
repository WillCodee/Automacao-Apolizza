## PRD
PRD-001 — Apolizza CRM (EPIC-003: Controle de Tarefas Diárias)

## Story
Como **gestor da corretora**, quero criar e gerenciar tarefas diárias para os cotadores, para organizar as atividades e garantir que nada seja esquecido.

## Acceptance Criteria

- [ ] AC1: Tabela `tarefas` criada com campos: id, titulo, descricao, data_vencimento, status, cotador_id, criador_id, created_at, updated_at
- [ ] AC2: Migration aplicada no Neon PostgreSQL
- [ ] AC3: API route `POST /api/tarefas` (criar tarefa — apenas admin/gestor)
- [ ] AC4: API route `GET /api/tarefas` (listar tarefas — gestor vê todas, cotador vê apenas as suas)
- [ ] AC5: API route `PATCH /api/tarefas/[id]` (editar tarefa — apenas admin/gestor)
- [ ] AC6: API route `DELETE /api/tarefas/[id]` (deletar tarefa — apenas admin/gestor)
- [ ] AC7: Validação de permissões: gestor pode CRUD completo, cotador apenas visualiza suas tarefas
- [ ] AC8: Interface em `/tarefas` com listagem de tarefas
- [ ] AC9: Formulário de criação de tarefa (título, descrição, data vencimento, cotador atribuído)
- [ ] AC10: Zod schemas para validação de inputs (createTarefaSchema, updateTarefaSchema)
- [ ] AC11: Testes unitários para API routes (create, read, update, delete)
- [ ] AC12: Foreign keys: cotador_id → users.id, criador_id → users.id
