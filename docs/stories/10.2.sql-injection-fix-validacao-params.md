# Story 10.2 — Fix SQL Injection e Validacao de Parametros

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-002 — Apolizza CRM v2 (Epic 1: Hardening & Seguranca)

## Story
Como **admin do sistema**, quero que todos os parametros de query sejam validados contra listas de valores permitidos antes de chegar ao SQL, para prevenir ataques de SQL injection e dados invalidos.

## Acceptance Criteria

- [x] AC1: Parametro `mes` no `/api/dashboard` e validado contra `MES_OPTIONS` antes do SQL
- [x] AC2: Parametro `ano` e validado como inteiro entre 2020 e 2030
- [x] AC3: Parametro `status` no `/api/cotacoes` e validado contra `STATUS_OPTIONS`
- [x] AC4: Parametro `assignee` e validado como UUID valido
- [x] AC5: Parametros invalidos retornam `400 Bad Request` com mensagem clara
- [x] AC6: Nenhum parametro de query string e interpolado em SQL sem validacao previa
