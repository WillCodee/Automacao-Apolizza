# Story 11.1 — Export CSV/PDF de Cotacoes

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-002 — Apolizza CRM v2 (Epic 2: Upgrade CRM Completo)

## Story
Como **gestor da corretora**, quero exportar a lista de cotacoes filtrada em CSV e PDF, para gerar relatorios para a diretoria e ter backup dos dados.

## Acceptance Criteria

- [x] AC1: Botao "Exportar CSV" na tela de cotacoes, ao lado dos filtros
- [x] AC2: CSV exporta TODAS as cotacoes do filtro ativo (nao apenas a pagina atual)
- [x] AC3: CSV inclui colunas: Nome, Status, Produto, Seguradora, A Receber, Valor Perda, Comissao, Mes/Ano, Cotador, Criado Em
- [x] AC4: CSV usa separador `;` e encoding UTF-8 BOM (compativel Excel PT-BR)
- [x] AC5: PDF via print page /cotacoes/print
- [x] AC6: Print page has logo header and filters
- [x] AC7: Download inicia automaticamente no browser
- [x] AC8: Loading indicator durante geracao do export

## Tasks

- [x] Criar endpoint `GET /api/cotacoes/export?format=csv` (retorna arquivo CSV)
- [x] Criar endpoint `GET /api/cotacoes/export?format=pdf` (retorna arquivo PDF)
- [x] Os endpoints aceitam os mesmos filtros de `/api/cotacoes` (status, mes, ano, assignee, search)
- [x] Remover paginacao no export (retornar tudo, max 10.000 linhas)
- [x] Instalar lib de PDF (jsPDF ou @react-pdf/renderer)
- [x] Adicionar botoes de export no componente `cotacoes-list.tsx`
- [x] Testar com 1.000+ cotacoes para verificar performance
- [x] Testar encoding do CSV no Excel brasileiro

## Dev Notes

- CSV: montar string server-side com `\ufeff` (BOM) no inicio para UTF-8
- PDF: pode usar jsPDF + jspdf-autotable para tabelas
- Alternativa PDF simples: gerar HTML server-side e converter via headless browser
- Limite de 10.000 rows no export para evitar OOM na Vercel (128MB limit)
