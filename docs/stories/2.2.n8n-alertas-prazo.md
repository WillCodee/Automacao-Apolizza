# Story 2.2 — N8N Workflows de Alertas de Prazo

## Status
InProgress

## Executor
@dev (Dex)

## Story
Como **cotador da corretora**, quero receber alertas no celular sobre prazos críticos (seguro fora do prazo, fim de vigência próximo, prazo diário às 15h/16h, próxima tratativa), para nunca perder um follow-up com cliente.

## Acceptance Criteria

- [x] AC1: Workflow N8N "Alerta Prazo Diário" roda às 15h e 16h (BRT) — lista tasks com due_date = hoje e status != "fechado"
- [x] AC2: Workflow N8N "Fim de Vigência" roda diariamente — lista tasks com campo "Fim Vigência" nos próximos 7 dias
- [x] AC3: Workflow N8N "Seguro Fora do Prazo" roda de hora em hora — detecta tasks com due_date ultrapassada e status != "fechado"/"atrasado"
- [x] AC4: Workflow N8N "Próxima Tratativa" roda diariamente — lista tasks com campo "Próxima Tratativa" = hoje
- [ ] AC5: Alertas enviados via canal definido pelo cliente (Telegram, WhatsApp ou email)
- [x] AC6: Mensagem do alerta inclui: nome da task, responsável, data de vencimento, link direto para o ClickUp

## Tasks

- [ ] Definir canal de notificação com o cliente (Telegram bot / WhatsApp via N8N / Email)
- [x] Criar workflow N8N "Alerta Prazo Diário 15h/16h" com Schedule Trigger + ClickUp API
- [x] Criar workflow N8N "Fim de Vigência" com Schedule Trigger diário
- [x] Criar workflow N8N "Seguro Fora do Prazo" com Schedule Trigger horário
- [x] Criar workflow N8N "Próxima Tratativa" com Schedule Trigger diário

- [ ] Testar cada workflow com tasks reais

## Dev Notes

- N8N_API_KEY e N8N_WEBHOOK_URL devem estar configurados no `.env`
- ClickUp API para buscar tasks: `GET /api/v2/list/{LIST_ID}/task` com filtros
- Campos de data do ClickUp retornam timestamp em milliseconds
- Para Telegram: criar bot via BotFather, obter chat_id do grupo/pessoa
- Para WhatsApp: usar N8N node oficial (requer WhatsApp Business API) ou Twilio
- Branch: `feat/dashboard-v2-modulos`
