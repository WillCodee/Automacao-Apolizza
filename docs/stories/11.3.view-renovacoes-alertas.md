# Story 11.3 — View Dedicada de Renovacoes com Alertas

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-002 — Apolizza CRM v2 (Epic 2: Upgrade CRM Completo)

## Story
Como **gestor da corretora**, quero uma pagina dedicada a renovacoes com alertas visuais de vencimento (60/30/15 dias), para acompanhar renovacoes proativamente e nao perder clientes.

## Acceptance Criteria

- [x] AC1: Nova pagina `/renovacoes` acessivel no nav header para todos
- [x] AC2: Lista apenas cotacoes com `is_renovacao = true` OU com `fim_vigencia` proximo
- [x] AC3: Badge colorido por urgencia: vermelho (< 15 dias), amarelo (< 30 dias), verde (< 60 dias), cinza (> 60 dias)
- [x] AC4: KPIs no topo: Total renovacoes, Vencendo em 30 dias, Renovadas (fechado), Perdidas
- [x] AC5: Filtros: cotador, mes/ano, urgencia (< 15d, < 30d, < 60d)
- [x] AC6: Coluna "Dias para Vencer" calculada em tempo real
- [x] AC7: Click na renovacao abre a cotacao para edicao
- [x] AC8: Admin ve todas; cotador ve apenas as suas

## Tasks

- [ ] Criar endpoint `GET /api/renovacoes` com filtros de urgencia e vencimento
- [ ] Criar view SQL `vw_renovacoes` com calculo de dias_para_vencer
- [ ] Criar pagina `/renovacoes/page.tsx` (server component com auth check)
- [ ] Criar componente `RenovacoesList` com tabela e badges de urgencia
- [ ] Criar componente `RenovacoesKpis` com 4 cards
- [ ] Adicionar "Renovacoes" no nav do `app-header.tsx`
- [ ] Atualizar type `activePage` no AppHeader
- [ ] Testar com cotacoes de vencimento variado
