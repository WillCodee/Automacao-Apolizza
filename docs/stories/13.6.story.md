# Story 13.6 — Upload de Anexos (Vercel Blob)

## Status
Draft

## Executor
@dev (Dex)

## Quality Gate
@architect (Aria)

## PRD
PRD-001 — Apolizza CRM (EPIC-003: Controle de Tarefas Diárias)

## Story
Como **cotador**, quero anexar arquivos (evidências, documentos) às tarefas, para compartilhar informações relevantes com a equipe.

## Acceptance Criteria

- [ ] AC1: Tabela `tarefas_anexos` criada (id, tarefa_id, usuario_id, nome_arquivo, url_blob, tamanho, created_at)
- [ ] AC2: Migration aplicada no Neon
- [ ] AC3: API route `POST /api/tarefas/[id]/anexos` (upload via Vercel Blob SDK)
- [ ] AC4: API route `GET /api/tarefas/[id]/anexos` (listar anexos da tarefa)
- [ ] AC5: API route `DELETE /api/tarefas/[id]/anexos/[anexoId]` (apenas criador do anexo ou gestor)
- [ ] AC6: Componente de upload com drag-and-drop
- [ ] AC7: Validação: max 10MB por arquivo
- [ ] AC8: Validação: tipos permitidos (PDF, PNG, JPG, JPEG, DOCX, XLSX)
- [ ] AC9: Preview de imagens (thumbnail ou lightbox)
- [ ] AC10: Links de download para PDFs e documentos
- [ ] AC11: Integração com histórico: ANEXO_ADICIONADO, ANEXO_REMOVIDO (Story 13.5)
- [ ] AC12: Loading state durante upload
- [ ] AC13: Error handling: arquivo muito grande, tipo não permitido, erro de upload

## Dev Notes

- **Vercel Blob:** Usar `put()` para upload, `del()` para deleção
- **MIME Types:** Usar `file.type` do FormData ou library como `file-type`
- **Drag-and-Drop:** `react-dropzone` ou native HTML5 drag-and-drop
- **Security:** Sanitizar nome do arquivo antes de salvar (evitar path traversal)
- **Storage:** Vercel Blob tem quota (verificar plano Vercel)
- **Branch:** `feat/epic-003-upload-anexos`
- **Quality Gate Tools:** code_review, security_scan, file_upload_validation
