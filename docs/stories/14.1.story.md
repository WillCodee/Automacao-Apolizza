# Story 14.1 — API dedicada para Painel TV

**Epic:** 14 — Painel TV Dashboard
**PRD:** PRD-011
**Status:** Ready
**Complexidade:** Simples

## Descricao

Criar endpoint `GET /api/tv?token=SECRET` que retorna todos os dados necessarios para o painel TV em uma unica chamada, sem exigir autenticacao via session — apenas validacao por token.

## Criterios de Aceitacao

- [ ] Endpoint `GET /api/tv` criado
- [ ] Validacao de token via query param `token` contra env `TV_TOKEN`
- [ ] Token invalido ou ausente retorna 401
- [ ] Resposta inclui: kpis, statusBreakdown, monthlyTrend, cotadores, metaMensal, semanas
- [ ] Dados do mes/ano atual (auto-detectado)
- [ ] Reutiliza queries existentes do `/api/dashboard` e `/api/dashboard/semanal`
- [ ] Normaliza campos numericos (DECIMAL → number)
- [ ] Responde em < 2s

## Detalhes Tecnicos

```typescript
// GET /api/tv?token=SECRET
// Response: { success: true, data: { kpis, statusBreakdown, monthlyTrend, cotadores, metaMensal, semanas } }
```

- Sem `getCurrentUser()` — autenticacao por token apenas
- Usa `dbQuery` para views SQL (vw_kpis, vw_status_breakdown, vw_cotadores, vw_monthly_trend)
- Usa Drizzle ORM para tabela `metas`
- Mes atual calculado automaticamente (MES_ARR + new Date())
- Correcao `mes_referencia` ja aplicada (usa `mesFullName`)

## Env

```
TV_TOKEN=<token-secreto-32-chars>
```
