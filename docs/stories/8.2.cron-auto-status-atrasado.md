# Story 8.2 — Cron Job Interno: Auto-Status ATRASADO

## Status
Draft

## Executor

## PRD
PRD-001 — Apolizza CRM (Fase 3: Automacoes + Polish)

## Story
Como **gestor da corretora**, quero que cotacoes com prazo vencido sejam automaticamente movidas para status "atrasado" por um job interno do sistema, para nao depender do N8N para essa regra critica.

## Acceptance Criteria

- [ ] AC1: Cron job roda a cada 1 hora (ou via Vercel Cron)
- [ ] AC2: Identifica cotacoes com `due_date < NOW()` e `status NOT IN ('fechado', 'perda', 'atrasado', 'cancelado')`
- [ ] AC3: Atualiza status para "atrasado" automaticamente
- [ ] AC4: Registra a mudanca na tabela `cotacao_history` (audit trail)
- [ ] AC5: Endpoint `POST /api/cron/auto-atrasado` protegido por API key (CRON_SECRET)
- [ ] AC6: Log de quantas cotacoes foram atualizadas em cada execucao
- [ ] AC7: Nao processa cotacoes sem `due_date` definida

## Tasks

- [ ] Criar `src/app/api/cron/auto-atrasado/route.ts`
- [ ] Implementar query SQL para encontrar cotacoes atrasadas
- [ ] Implementar UPDATE em batch
- [ ] Registrar cada mudanca em `cotacao_history`
- [ ] Proteger endpoint com `CRON_SECRET` header validation
- [ ] Configurar `vercel.json` com cron schedule (ou usar N8N como trigger)
- [ ] Testar com cotacoes de teste (due_date no passado)
- [ ] Monitorar primeiras execucoes em producao

## Dev Notes

- **Vercel Cron:** `vercel.json` → `{ "crons": [{ "path": "/api/cron/auto-atrasado", "schedule": "0 * * * *" }] }`
- **Seguranca:** Vercel envia header `Authorization: Bearer {CRON_SECRET}`. Validar no endpoint
- **Free tier Vercel:** Cron limitado a 1 execucao/dia no free. Se precisar de mais, manter N8N
- **Alternativa:** Manter o workflow N8N existente (Story 8.1) e remover este cron
- **Audit trail:** Cada mudanca gera registro com `field_name: 'status'`, `old_value`, `new_value: 'atrasado'`, `user_id: null` (sistema)
- **Depende de:** Story 5.1 (schema), Story 8.3 (cotacao_history)
