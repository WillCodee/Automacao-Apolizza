# Story 9.2 — Remover Codigo ClickUp do Sistema

## Status
Draft

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 4: Desligamento ClickUp)

## Story
Como **desenvolvedor**, quero remover todo o codigo de integracao com o ClickUp do sistema, para eliminar a dependencia e simplificar a codebase.

## Acceptance Criteria

- [ ] AC1: Removidos todos os endpoints de proxy ClickUp (`server.py` pode ser deletado)
- [ ] AC2: Removidas todas as constantes FIELD_* com UUIDs do ClickUp no frontend
- [ ] AC3: Removidas funcoes `fetchAllTasks()` e `fetchRenovacaoTasks()` que chamam ClickUp API
- [ ] AC4: Removido token ClickUp de qualquer arquivo de configuracao
- [ ] AC5: Removidos dados PRELOADED hardcoded (fallback offline do ClickUp)
- [ ] AC6: `dashboard-legacy.html` removido do `/public` (se ainda existir)
- [ ] AC7: N8N workflows nao fazem mais chamadas para ClickUp API
- [ ] AC8: `.env.example` atualizado removendo `CLICKUP_API_KEY`
- [ ] AC9: Nenhuma referencia a "clickup" restante no codigo (exceto `clickup_id` na tabela de migracao)
- [ ] AC10: Sistema funciona 100% sem ClickUp

## Tasks

- [ ] Deletar `server.py` (proxy Python)
- [ ] Deletar `server.ps1` e `iniciar.bat` (scripts de inicializacao do proxy)
- [ ] Remover constantes FIELD_*, TOKEN, CLICKUP_API do codigo
- [ ] Remover funcoes de fetch do ClickUp
- [ ] Remover dados PRELOADED
- [ ] Remover `dashboard-legacy.html` se existir
- [ ] Atualizar `.env.example` removendo CLICKUP_API_KEY
- [ ] Verificar N8N workflows nao usam mais ClickUp
- [ ] Rodar busca por "clickup" em todo o projeto — garantir zero referencias
- [ ] Testar sistema completo apos remocao

- **Dashboard legacy:** Se a migracao para React estiver completa, deletar. Senao, manter temporariamente
