# Story 7.3 — Metas Persistidas no Banco de Dados

## Status
Draft

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 2: Dashboard Proprio)

## Story
Como **gestor da corretora**, quero que as metas da corretora e dos cotadores sejam salvas no banco de dados (nao mais em localStorage), para que todos os usuarios vejam as mesmas metas e elas nao se percam ao trocar de navegador.

## Acceptance Criteria

- [ ] AC1: Endpoint `GET /api/metas` retorna metas filtradas por ano/mes
- [ ] AC2: Endpoint `POST /api/metas` cria meta (global ou individual)
- [ ] AC3: Endpoint `PUT /api/metas/[id]` atualiza meta existente
- [ ] AC4: Meta global (user_id = NULL) acessivel por todos
- [ ] AC5: Meta individual (user_id = cotador) acessivel pelo cotador e admins
- [ ] AC6: Somente admin pode criar/editar metas
- [ ] AC7: View de Metas no dashboard carrega dados do banco
- [ ] AC8: Barras de progresso calculam % atingimento comparando meta vs realizado (via view SQL)
- [ ] AC9: Seletor de mes funciona com dados do banco
- [ ] AC10: Cards de cotador mostram progresso vs meta individual
- [ ] AC11: Dados de localStorage migrados para o banco (one-time migration)
