# Story 11.10 — Operacoes em Lote e Importacao CSV

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-002 — Apolizza CRM v2 (Epic 2: Upgrade CRM Completo)

## Story
Como **admin da corretora**, quero poder alterar status de multiplas cotacoes de uma vez e importar cotacoes via CSV, para agilizar operacoes em lote e facilitar carga de dados.

## Acceptance Criteria

### Operacoes em Lote
- [x] AC1: Checkboxes de selecao na tabela de cotacoes (individual + selecionar todos da pagina)
- [x] AC2: Barra de acoes aparece ao selecionar 1+ cotacoes
- [x] AC3: Acao "Alterar Status" abre dropdown com status disponiveis
- [x] AC4: Acao "Excluir Selecionados" com confirmacao (admin only)
- [x] AC5: Validacao de campos obrigatorios para status terminal aplicada em lote
- [x] AC6: Feedback: "X de Y atualizados com sucesso, Z falharam (campos obrigatorios)"
- [x] AC7: Audit trail gerado para cada cotacao alterada

### Importacao CSV
- [x] AC8: Botao "Importar CSV" na pagina de cotacoes (admin only)
- [x] AC9: Upload de arquivo .csv com preview das primeiras 5 linhas
- [x] AC10: Mapeamento de colunas: usuario confirma qual coluna CSV corresponde a qual campo
- [x] AC11: Validacao pre-importacao: mostra erros por linha (campo invalido, duplicata)
- [x] AC12: Importacao cria cotacoes com assignee_id do admin logado (ou mapeado)
- [x] AC13: Resultado: "X importadas, Y ignoradas (erros)"
- [x] AC14: Limite de 1.000 linhas por importacao

## Tasks

### Bulk Operations
- [x] Criar endpoint `POST /api/cotacoes/bulk` com body `{ ids: [], action: "updateStatus", data: { status } }`
- [x] Implementar logica de bulk status update com transacao DB
- [x] Implementar logica de bulk soft delete com transacao DB
- [x] Gerar audit trail em batch para todas as cotacoes afetadas
- [x] Adicionar checkboxes na tabela de cotacoes
- [x] Criar componente `BulkActionBar` com contagem e botoes

### Importacao CSV
- [x] Criar endpoint `POST /api/cotacoes/import` que recebe FormData com arquivo CSV
- [x] Implementar parser CSV server-side (nao depender de lib — csv simples)
- [x] Criar componente `CsvImportModal` com upload, preview e mapeamento
- [x] Implementar validacao Zod em cada linha antes do insert
- [x] Inserir em batch com transacao (rollback se > 10% de erros)
- [x] Retornar resultado detalhado (sucesso/erro por linha)

## Dev Notes

- Bulk: usar `WHERE id IN (...)` com transacao
- CSV parser: `text.split('\n').map(line => line.split(';'))` para CSV brasileiro
- Encoding: detectar UTF-8 BOM e tratar corretamente
- Limite de 1.000 linhas = ~100KB CSV — bem dentro do limit do Vercel
