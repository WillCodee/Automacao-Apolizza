# Story 7.2 — KPIs Calculados por Views SQL

## Status
Draft

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 2: Dashboard Proprio)

## Story
Como **gestor da corretora**, quero que os KPIs e metricas de desempenho sejam calculados diretamente no banco de dados via views SQL otimizadas, para ter respostas rapidas e precisas sem calculo no frontend.

## Acceptance Criteria

- [ ] AC1: View `vw_kpis_mensal` retorna: qtd_fechadas, qtd_perda, total_cotacoes, total_a_receber, total_valor_perda, taxa_conversao — agrupado por mes
- [ ] AC2: View `vw_desempenho_cotador` retorna: total_cotacoes, fechadas, perdas, faturamento, taxa_conversao — por cotador
- [ ] AC3: Endpoint `GET /api/kpis` usa a view `vw_kpis_mensal` com filtros de ano/mes
- [ ] AC4: Endpoint `GET /api/kpis/cotadores` retorna dados da view `vw_desempenho_cotador`
- [ ] AC5: Tempo de resposta dos endpoints de KPI < 200ms
- [ ] AC6: Indices criados nas colunas mais filtradas: status, due_date, assignee_id, ano_referencia
- [ ] AC7: Dashboard usa endpoints de KPI ao inves de calcular no client
- [ ] AC8: Dados do dashboard de desempenho por cotador vem da view SQL

## Tasks

- [ ] Criar migration com views SQL (`vw_kpis_mensal`, `vw_desempenho_cotador`)
- [ ] Criar indices: `idx_cotacoes_status`, `idx_cotacoes_due_date`, `idx_cotacoes_assignee`, `idx_cotacoes_ano`
- [ ] Criar endpoint `GET /api/kpis` que consulta a view com filtros
- [ ] Criar endpoint `GET /api/kpis/cotadores` que consulta a view de desempenho
- [ ] Medir tempo de resposta com 3.279+ registros
- [ ] Otimizar queries se necessario (EXPLAIN ANALYZE)
- [ ] Integrar endpoints no dashboard frontend

## Dev Notes

- **Views SQL:** Definidas no PRD secao 4.2 — usar exatamente as queries la
- **Indices:** Essenciais para performance com 3k+ registros. Criar via migration Drizzle
- **Drizzle nao gera views:** Criar via migration SQL customizada (`sql` tag do Drizzle)
- **Materialized views:** Se performance for um problema, considerar `MATERIALIZED VIEW` com refresh periodico. Provavelmente desnecessario com 3k registros
