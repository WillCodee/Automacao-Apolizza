# Story 6.5 — Execucao da Migracao e Validacao de Integridade

## Status
Draft

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 1: CRUD + Migracao)

## Story
Como **gestor da corretora**, quero que todas as 3.279+ cotacoes sejam migradas do ClickUp para o Neon com 100% de integridade, para confiar que o novo sistema tem os mesmos dados do antigo.

## Acceptance Criteria

- [ ] AC1: Migracao executada com sucesso (0 erros)
- [ ] AC2: Contagem de cotacoes no Neon = contagem no ClickUp
- [ ] AC3: Soma de `a_receber` no Neon = soma de FIELD_A_RECEBER no ClickUp (para status fechado)
- [ ] AC4: Soma de `valor_perda` no Neon = soma de FIELD_VALOR_PERDA no ClickUp (para status perda)
- [ ] AC5: Todos os assignees do ClickUp existem como users no Neon
- [ ] AC6: Contagem por status no Neon = contagem por status no ClickUp
- [ ] AC7: Renovacoes migradas com `is_renovacao = true`
- [ ] AC8: Relatorio de validacao gerado com comparativo lado a lado
- [ ] AC9: Backup do JSON extraido do ClickUp salvo em `data/clickup-backup-{date}.json`

## Tasks

- [ ] Rodar script de migracao (Story 6.4) em ambiente de producao
- [ ] Criar script `scripts/validate-migration.ts` que compara contagens e somas
- [ ] Comparar totais: cotacoes, por status, por assignee
- [ ] Comparar somas financeiras: a_receber, valor_perda
- [ ] Verificar campos individuais em amostra de 20 cotacoes
- [ ] Salvar backup JSON completo do ClickUp
- [ ] Gerar relatorio markdown com resultados
- [ ] Corrigir quaisquer discrepancias encontradas

## Dev Notes

- **Validacao critica:** Este e o ponto mais importante da migracao. Se os numeros nao baterem, nao avancamos
- **Comparacao:** O script deve consultar tanto a API do ClickUp quanto o Neon e comparar lado a lado
- **Backup:** Salvar o JSON raw do ClickUp como seguranca — nunca se sabe quando vai precisar
- **Amostra manual:** Escolher 20 cotacoes aleatorias e verificar campo a campo
- **Depende de:** Story 6.4 (script de migracao)
