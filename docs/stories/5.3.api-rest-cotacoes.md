# Story 5.3 — API Base: Endpoints REST para Cotacoes

## Status
Done

## Executor

## PRD
PRD-001 — Apolizza CRM (Fase 0: Fundacao)

## Story
Como **frontend do CRM**, quero endpoints REST completos para CRUD de cotacoes, para que o dashboard e formularios possam criar, ler, atualizar e listar cotacoes do banco Neon.

## Acceptance Criteria

- [x] AC1: `GET /api/cotacoes` retorna lista paginada de cotacoes (default 50/page)
- [x] AC2: `GET /api/cotacoes` aceita query params: `page`, `status`, `ano`, `mes`, `assignee`, `search`
- [x] AC3: Cotador autenticado recebe apenas suas cotacoes; admin recebe todas
- [x] AC4: `GET /api/cotacoes/[id]` retorna uma cotacao completa com todos os campos
- [x] AC5: `POST /api/cotacoes` cria nova cotacao com validacao Zod
- [x] AC6: `PUT /api/cotacoes/[id]` atualiza cotacao existente (partial update)
- [x] AC7: `DELETE /api/cotacoes/[id]` faz soft-delete (somente admin)
- [x] AC8: `GET /api/kpis` retorna KPIs agregados: a_receber, valor_perda, total, taxa_conversao
- [x] AC9: `GET /api/kpis` aceita filtros: ano, mes, assignee
- [x] AC10: Todos os endpoints retornam JSON padronizado com `{ data, error, pagination }`
- [x] AC11: Todos os endpoints validam autenticacao via `getCurrentUser()`
- [x] AC12: Valores monetarios retornam como number (nao string)
- [x] AC13: Timestamps retornam em ISO 8601

## Dev Notes

- **Paginacao:** Query-based com `?page=1&limit=50`. Header `X-Total-Count` opcional
- **Search:** `WHERE name ILIKE '%termo%'` — simples e suficiente para 3k+ registros
- **Validacao:** Usar Zod para validar body do POST/PUT. Schema Zod deve espelhar o Drizzle schema
- **Soft delete:** `UPDATE cotacoes SET deleted_at = NOW() WHERE id = ?`. Queries de listagem filtram `WHERE deleted_at IS NULL`
- **KPIs endpoint:** Usar a view `vw_kpis_mensal` ou query equivalente com filtros
