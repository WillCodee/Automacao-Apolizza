# Story 10.3 — Validacao Zod Completa com Enums

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-002 — Apolizza CRM v2 (Epic 1: Hardening & Seguranca)

## Story
Como **admin do sistema**, quero que todos os campos de cotacao sejam validados contra enums e formatos definidos, para impedir dados invalidos no banco e garantir integridade.

## Acceptance Criteria

- [x] AC1: Campo `status` e validado contra `STATUS_OPTIONS` (12 valores)
- [x] AC2: Campo `priority` e validado contra `PRIORITY_OPTIONS`
- [x] AC3: Campo `tipoCliente` e validado contra `TIPO_CLIENTE_OPTIONS`
- [x] AC4: Campo `produto` e validado contra `PRODUTO_OPTIONS` (46 valores)
- [x] AC5: Campo `situacao` e validado contra `SITUACAO_OPTIONS`
- [x] AC6: Campo `mesReferencia` e validado contra `MES_OPTIONS`
- [x] AC7: Campo `anoReferencia` e validado como inteiro entre 2020 e 2030
- [x] AC8: Campo `parceladoEm` e validado como inteiro entre 1 e 120
- [ ] AC9: Campos monetarios (premioSemIof, comissao, aReceber, valorPerda) aceitam apenas valores >= 0
- [x] AC10: Endpoint POST /api/cotacoes rejeita payload com valores fora dos enums (422)
- [x] AC11: Endpoint PUT /api/cotacoes/[id] aplica as mesmas validacoes
- [ ] AC12: Schema de criacao de usuario valida email (formato) e senha (min 8 chars)
