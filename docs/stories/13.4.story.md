# Story 13.4 — Notificações Automáticas (Vercel Cron + Resend)

## Status
Draft

## Executor

## Quality Gate

## PRD
PRD-001 — Apolizza CRM (EPIC-003: Controle de Tarefas Diárias)

## Story
Como **cotador**, quero receber notificações por email sobre tarefas novas, atrasadas ou concluídas, para não esquecer dos meus compromissos.

## Acceptance Criteria

- [ ] AC1: Vercel Cron job configurado em `vercel.json` (executa diariamente às 8h)
- [ ] AC2: API route `/api/cron/tarefas-notificacoes` implementada
- [ ] AC3: Template HTML de email para "Nova Tarefa" (inclui título, descrição, data vencimento)
- [ ] AC4: Template HTML de email para "Tarefa Atrasada" (enviado para cotador + gestor)
- [ ] AC5: Template HTML de email para "Tarefa Concluída" (enviado para gestor)
- [ ] AC6: Integração com Resend API (env var `RESEND_API_KEY`)
- [ ] AC7: Lógica de envio: nova tarefa → email para cotador atribuído
- [ ] AC8: Lógica de envio: tarefa atrasada → email para cotador + gestor (se `data_vencimento < NOW()` e status != Concluída)
- [ ] AC9: Lógica de envio: tarefa concluída → email para gestor (se status mudou para Concluída hoje)
- [ ] AC10: Logs de envio (success/failure) em console e/ou tabela `email_logs`
- [ ] AC11: Rate limiting para evitar spam (máx 1 email por tarefa por dia)
- [ ] AC12: Teste manual: criar tarefa, verificar email recebido

## Tasks

- [ ] Adicionar cron schedule em `vercel.json`: `"crons": [{"path": "/api/cron/tarefas-notificacoes", "schedule": "0 8 * * *"}]`
- [ ] Criar API route `POST /api/cron/tarefas-notificacoes/route.ts`
- [ ] Instalar `resend` SDK: `npm install resend`
- [ ] Criar template HTML `nova-tarefa-email.tsx` em `apolizza-crm/src/lib/email-templates/`
- [ ] Criar template HTML `tarefa-atrasada-email.tsx`
- [ ] Criar template HTML `tarefa-concluida-email.tsx`
- [ ] Implementar lógica de query: `SELECT * FROM tarefas WHERE created_at::date = NOW()::date` (novas)
- [ ] Implementar lógica de query: `SELECT * FROM tarefas WHERE data_vencimento < NOW() AND status != 'Concluída'` (atrasadas)
- [ ] Implementar lógica de query: `SELECT * FROM tarefas WHERE updated_at::date = NOW()::date AND status = 'Concluída'` (concluídas hoje)
- [ ] Implementar função `sendEmailNotification(to, subject, html)`
- [ ] Adicionar tratamento de erro: log failure, não bloquear cron
- [ ] Adicionar env var `RESEND_API_KEY` no `.env.local` e Vercel dashboard
- [ ] Testar localmente com `npm run dev` + trigger manual
- [ ] Testar em staging/preview deploy

## Dev Notes

- **Resend:** Usar domínio verificado (ex: `notificacoes@apolizza.com` ou Resend sandbox)
- **Cron Schedule:** `"0 8 * * *"` = todo dia às 8h UTC (ajustar timezone se necessário)
- **Rate Limiting:** Criar tabela `email_logs` (opcional) ou usar flag `email_enviado_em` na tarefa
- **Error Handling:** Não lançar exceção se Resend falhar — apenas logar
- **Branch:** `feat/epic-003-notificacoes`
- **Quality Gate Tools:** code_review, integration_test, secrets_validation
