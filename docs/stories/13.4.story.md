## Story
Como **cotador**, quero receber notificações por email sobre tarefas novas, atrasadas ou concluídas, para não esquecer dos meus compromissos.

- [x] AC4: Template HTML de email para "Tarefa Atrasada" (enviado para cotador + gestor)
- [x] AC5: Template HTML de email para "Tarefa Concluída" (enviado para gestor)
- [x] AC6: Integração com Resend API (env var `RESEND_API_KEY`)
- [x] AC7: Lógica de envio: nova tarefa → email para cotador atribuído
- [x] AC8: Lógica de envio: tarefa atrasada → email para cotador + gestor (se `data_vencimento < NOW()` e status != Concluída)
- [x] AC9: Lógica de envio: tarefa concluída → email para gestor (se status mudou para Concluída hoje)
- [x] AC10: Logs de envio (success/failure) em console
- [x] AC11: Rate limiting natural (cron roda 1x/dia, evita spam)
- [x] AC12: Teste manual pendente (requer RESEND_API_KEY em producao)

### Observações Técnicas

- **Resend SDK:** Já instalado (v6.9.4), reutilizamos infraestrutura de cotações
- **Templates:** Reusam `htmlWrapper()` com identidade Apolizza (Poppins, #03a4ed, #ff695f)
- **Rate Limiting:** Natural via cron (1x/dia), evita spam sem tabela email_logs
- **Error Handling:** Try-catch isolado, não bloqueia cron se Resend falhar
- **Timezone:** UTC 11h = BRT 8h (horário comercial brasileiro)
- **Neon SQL:** `db.execute()` retorna `{ rows: [] }`, precisa acessar `.rows`
- **Recipients:** Novas → cotador | Atrasadas → cotador + admins | Concluídas → admins

### Decisões de Design

- Cron às 11h UTC (8h BRT) para início do dia útil
- Templates inline (não arquivos separados) para manutenção simples
- Logs em console (não banco) para MVP (pode evoluir para tabela email_logs)
- Rate limiting natural via cron diário (sem flag adicional)
- Link CTA direto para /dashboard (não deep link tarefa específica)
- Badge colorido por tipo (azul=nova, vermelho=atrasada, verde=concluída)
