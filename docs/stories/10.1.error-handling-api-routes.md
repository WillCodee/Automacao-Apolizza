# Story 10.1 — Error Handling Padronizado em Todas API Routes

## Status
Done

## Executor
@dev (Dex)

## PRD
PRD-002 — Apolizza CRM v2 (Epic 1: Hardening & Seguranca)

## Story
Como **desenvolvedor do sistema**, quero que todas as API routes tenham try-catch padronizado com logging e respostas consistentes, para evitar vazamento de stack trace e garantir resiliencia em producao.

## Acceptance Criteria

- [x] AC1: Todas as 14 API route handlers (GET/POST/PUT/DELETE) estao envoltas em try-catch
- [x] AC2: Erros retornam `apiError("mensagem em portugues", 500)` sem vazar detalhes internos
- [x] AC3: Todos os erros sao logados com `console.error` incluindo rota e metodo
- [x] AC4: Erros de validacao Zod retornam 422 com mensagens claras
- [x] AC5: Erros de "nao encontrado" retornam 404
- [x] AC6: Erros de autenticacao retornam 401
- [x] AC7: Erros de autorizacao retornam 403
- [x] AC8: Nenhuma rota retorna o stack trace padrao do Next.js em producao
