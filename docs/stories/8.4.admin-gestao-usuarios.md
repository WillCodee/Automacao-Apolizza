# Story 8.4 — Admin: Gestao de Usuarios

## Status
Draft

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 3: Automacoes + Polish)

## Story
Como **admin da corretora**, quero criar, editar e desativar contas de cotadores no sistema, para gerenciar a equipe sem precisar de acesso ao banco de dados.

## Acceptance Criteria

- [ ] AC1: Pagina `/admin/users` acessivel somente por admins
- [ ] AC2: Lista de todos os usuarios com nome, email, role, status (ativo/inativo)
- [ ] AC3: Botao "Novo Usuario" abre formulario de criacao
- [ ] AC4: Formulario de criacao: nome, email, username, senha, role (admin/cotador), foto URL
- [ ] AC5: Senha armazenada com bcrypt (salt rounds >= 12)
- [ ] AC6: Botao "Editar" permite alterar nome, email, foto, role
- [ ] AC7: Botao "Desativar" faz soft-disable (is_active = false)
- [ ] AC8: Usuario desativado nao consegue fazer login
- [ ] AC9: Admin nao pode desativar a si mesmo
- [ ] AC10: Endpoint `GET /api/admin/users` retorna lista (somente admin)
- [ ] AC11: Endpoint `POST /api/admin/users` cria usuario (somente admin)
- [ ] AC12: Endpoint `PUT /api/admin/users/[id]` atualiza usuario (somente admin)

## Tasks

- [ ] Criar `src/app/api/admin/users/route.ts` (GET list + POST create)
- [ ] Criar `src/app/api/admin/users/[id]/route.ts` (PUT update)
- [ ] Implementar middleware/guard de admin em todos os endpoints
- [ ] Criar pagina `src/app/admin/users/page.tsx` com lista de usuarios
- [ ] Criar componente `UserForm` (criacao e edicao)
- [ ] Implementar logica de desativacao (is_active = false)
- [ ] Implementar hash bcrypt na criacao de usuario
- [ ] Adicionar validacao: email unico, username unico
- [ ] Testar criacao, edicao e desativacao
- [ ] Testar que cotador nao acessa `/admin/users`

## Dev Notes

- **Guarda de admin:** Usar `requireAdmin()` helper (Story 5.2) em todos os endpoints
- **Senha:** Hash com bcrypt no backend. Nunca armazenar plaintext
- **Desativacao:** Soft-disable — usuario continua no banco mas nao faz login. Auth.js deve checar `is_active` no authorize()
- **Auto-desativacao:** Prevenir admin de desativar sua propria conta
- **Depende de:** Story 5.1 (tabela users), Story 5.2 (auth + requireAdmin)
