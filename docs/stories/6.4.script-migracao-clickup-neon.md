# Story 6.4 — Script de Migracao ClickUp para Neon

## Status
Draft

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 1: CRUD + Migracao)

## Story
Como **desenvolvedor**, quero um script de migracao que extraia todas as cotacoes do ClickUp e as insira no banco Neon, para ter os dados historicos disponiveis no novo sistema.

## Acceptance Criteria

- [ ] AC1: Script extrai todas as tasks da lista de Cotacoes (ID: 900701916229) via ClickUp API v2
- [ ] AC2: Script extrai todas as tasks do Space de Renovacao (ID: 90070369721)
- [ ] AC3: Paginacao completa — busca todas as 33+ paginas (3.279+ tasks)
- [ ] AC4: Mapeia todos os 19 custom fields para as colunas correspondentes da tabela `cotacoes`
- [ ] AC5: Mapeia `task.assignees[0].username` para `users.id` (lookup por username)
- [ ] AC6: Cria users na tabela `users` a partir dos assignees unicos do ClickUp (role=cotador, senha temporaria)
- [ ] AC7: Normaliza status para lowercase
- [ ] AC8: Converte `due_date` de timestamp ms para TIMESTAMPTZ
- [ ] AC9: Converte valores de dropdown (orderindex) para labels textuais
- [ ] AC10: Tasks do Space de Renovacao recebem `is_renovacao = true`
- [ ] AC11: `clickup_id` preservado para rastreabilidade
- [ ] AC12: Script gera relatorio de migracao: total extraido, total inserido, erros
- [ ] AC13: Script e idempotente — pode rodar novamente sem duplicar (upsert por clickup_id)
- [ ] AC14: Script roda via `npx tsx scripts/migrate-clickup.ts`

## Tasks
