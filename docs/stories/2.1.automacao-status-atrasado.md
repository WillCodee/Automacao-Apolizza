# Story 2.1 — Automação de Status ATRASADO no ClickUp

## Status
InProgress

## Executor
@dev (Dex)

## Story
Como **gestor da corretora**, quero que tasks com prazo vencido e não concluídas sejam automaticamente movidas para o status "ATRASADO" com notificação ao responsável, para garantir que nenhuma cotação passe despercebida.

## Acceptance Criteria

- [x] AC1: Automação nativa do ClickUp configurada: quando `due_date` < hoje E status != "fechado" → mover para "atrasado"
- [ ] AC2: Responsável da task recebe notificação do ClickUp ao mudar para "atrasado"
- [x] AC3: Automação roda automaticamente (trigger por data, não manual)
- [x] AC4: Tasks já atrasadas no momento da ativação são processadas
- [x] AC5: Documentar configuração da automação para manutenção futura

## Tasks

- [x] Verificar automações nativas disponíveis no plano ClickUp da Apolizza
- [x] Configurar automação: workflow N8N com schedule trigger horário + ClickUp API PUT
- [ ] Configurar notificação ao assignee na mudança de status
- [ ] Testar com task de teste (criar, definir due_date no passado, verificar mudança)
- [x] Documentar passo a passo da configuração em `docs/automacoes/status-atrasado.md`

## Dev Notes

- Esta story é de **configuração no ClickUp**, não de código no dashboard
- Alternativa se automação nativa for limitada: N8N workflow com schedule trigger que consulta API do ClickUp e atualiza tasks
- ClickUp Automations: Settings > Automations > "When due date arrives" trigger
- Branch: `feat/dashboard-v2-modulos` (apenas docs)
