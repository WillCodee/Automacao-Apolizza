# Story 7.4 — Upload de Documentos (Vercel Blob)

## Status
Draft

## Executor
@dev (Dex)

## PRD
PRD-001 — Apolizza CRM (Fase 2: Dashboard Proprio)

## Story
Como **cotador**, quero anexar documentos (PDF, imagens) a uma cotacao, para manter toda a documentacao do seguro centralizada no sistema.

## Acceptance Criteria

- [ ] AC1: Botao "Anexar Documento" no formulario de edicao da cotacao
- [ ] AC2: Upload aceita PDF, JPG, PNG, DOCX (max 10MB por arquivo)
- [ ] AC3: Arquivo enviado para Vercel Blob via `PUT /api/cotacoes/[id]/docs`
- [ ] AC4: Registro criado na tabela `cotacao_docs` com file_name, file_url, file_size, mime_type
- [ ] AC5: Lista de documentos anexados exibida na pagina da cotacao
- [ ] AC6: Link para download/visualizacao de cada documento
- [ ] AC7: Botao de remover documento (somente quem enviou ou admin)
- [ ] AC8: Feedback visual de progresso durante upload
- [ ] AC9: Validacao de tipo e tamanho de arquivo antes do upload

## Tasks

- [ ] Instalar `@vercel/blob`
- [ ] Criar endpoint `POST /api/cotacoes/[id]/docs` para upload
- [ ] Criar endpoint `GET /api/cotacoes/[id]/docs` para listar documentos
- [ ] Criar endpoint `DELETE /api/cotacoes/[id]/docs/[docId]` para remover
- [ ] Implementar upload via `put()` do Vercel Blob SDK
- [ ] Criar componente `DocumentUpload` com drag-and-drop ou file input
- [ ] Criar componente `DocumentList` com links e botao de remover
- [ ] Adicionar validacao de tipo MIME e tamanho
- [ ] Adicionar barra de progresso visual
- [ ] Testar upload, listagem e remocao

## Dev Notes

- **Vercel Blob:** `import { put, del } from '@vercel/blob'` — SDK simples e direto
- **Free tier:** 256MB de storage. Suficiente para o volume atual
- **Organizacao:** Arquivos salvos como `docs/{cotacao_id}/{filename}` no Blob
- **BLOB_READ_WRITE_TOKEN:** Necessario no `.env.local` para o SDK funcionar
- **Depende de:** Story 5.1 (tabela cotacao_docs), Story 6.2 (formulario de edicao)
- Branch: `feat/apolizza-crm`
- PRD ref: PRD-001 secao 5.2 (FR-CRUD-07)
